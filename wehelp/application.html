自我介紹

我是洪暐銘，習慣大家叫我 Bruce 。10 年工作資歷，其中 8 年是負責軟體銷售業務，包含：美食外送服務、電商系統、行銷數據CDP、企業BPM系統，作爲業務主管帶領團隊時，曾為公司創造單月破千萬業績，YOY達170%的戰績。

歷經多年業績高峰低谷，厭倦收入不穩定、工作價值只能以業績高低來衡量的業務工作。熱愛學習新知的我，想繼續待在日新月異的軟體業，轉職為軟體工程師，累積技術硬底子，追求穩定收入和潛力高的職涯發展。

為了成為軟體工程師，曾做過什麼努力？若有具體專案作品請分享給我們。

  • Linkedin Learning（年費會員） - Python, HTML, CSS, JavaScript
  • MDN Web Doc

  • W3Schools Online Web Tutorial

    如果參與這個計畫，會怎麼安排學習時間？

    如果有機會加入 WeHelp，每日學習規劃如下：

  • 每天 9:00 - 9:30 復盤前一天學習狀況，如遇到自己無法解決的開發問題，至少花一天時間研究如何解決，無法自行解決，隔天詢問助教。
  • 每週一參加 10:00 線上會議，並於會議結束後，開始學習課程。
  • 週二～四 9:30 - 18:00 按照課表學習，並於 18:30 花 15 分鐘撰寫每日學習報告，花 15 分鐘複習當日學習重點。
  • 每週五，參加實體課程，與同儕交流，有問題當面請教老師、助教。
  • 每週六～日，10:00 - 16:00練習當週學習的程式語言，並且進行該階段的作業或專案內容，同時嘗試累積更多 side project。
    • 是否有想要加入的軟體公司？為什麼想加入該公司？
    想成為全端工程師（鑽研後端和 ios 應用程式開發），進入以開發「企業BPM」的軟體公司為目標，因為這類型公司規模小型大型都有，工作機會也多。
    請描述一件讓你產生明顯負面情緒的事情，你如何處理該情緒？

    在工作上遇到無法以解決問題為目的的合作夥伴，會讓我很沮喪。遇到這種狀況，先以解決問題為優先，接下來再處理彼此溝通的問題。

    舉例來說，過去我在負責大型客戶專案期間，常遇到功能客製和系統串接問題，經常需要和後端工程師與客戶開三方技術會議。與會的工程師並非擅長面對客戶，因此會不經意地回答會過於仔細，偶爾發生會議越開越長，從原本討論範圍內延伸更多新的需求。

    此時，在會議上面對客戶，我會把焦點強調本次合作最大的價值和優先解決的問題，除非現場客戶端的窗口有權限改變專案內容，三方討論應該聚焦在原本討論範圍，其他需求也很重要可以放在之後階段討論。開完會後，我會先和工程師討論專案排程是否能如預期交給客戶，最後再主動提出自己能夠改進的地方，讓彼此合作可以更好，通常能讓雙方引起良性的互動。

    關於這份申請網頁，請分享一個你開發時的技術心得

    很有成就感！發現學習和實際進開發完全是兩回事，尤其是 code 寫進去後，網頁呈現卻沒反應或是改動到不對的地方，才知道有觀念沒搞懂。唯一解決的方式，就是要有耐心先找出哪裡有問題，再思考如何調整！

    這是我第一次開發網頁，彭彭老師曾提過「寫HTML不難，難是難在網頁切版」，結果實際開發果然被我遇到，因就因為沒搞清楚「容器和項目」的階層概念，例如：要調整項目中的文字、背景顏色、行寬、行高，都要在上一層容器的CSS選擇器設定，當段落越多，就越容易搞混自己現在到底改到哪一層了？？？

    後來，反覆觀看彭彭老師和 Morten Rand-Hendriksen（Linkedin Learning的線上講師）講解網頁切版的課程，釐清切版的概念，同時找一個真實網站（WIRED）查看網頁原始碼編寫的架構，然後再回到VSC嘗試編寫，花了2天才解決問題。

    其他想要對我們說的事情？

    感謝彭彭老師在 Youtube 上提供免費的入門課程，對於完全沒有程式語言基礎、又想在 37 歲時轉職當工程師的我來說，在自學過程中增加了不少信心，覺得程式語言的越來越有趣了。回顧自己的業務職涯，週遭的同事都在追求創業當老闆或者賺更多的錢。反觀自己隨著年資越深，除了厭倦收入時高時低之外，在每次的案子，反而是越來越鑽研在協助客戶解決問題，而非單純追求成交賺獎金。因此，從一年前認真思考轉職工程師這條路。

    我擔心貿然地投入金錢和時間去上培訓班，會承擔過多的風險。於是先做一些功課瞭解這個職業，爬了不少文和看一些 Youtube 影片談論關於轉職工程師的心路歷程，也跟幾位工程師聊過，得到的反饋都是「這一份要持續學習、解決問題的工作，否則會很快失去競爭力」。為了確認自己是否有學習的意願和熱忱，從半年前，趁工作閒暇之餘，從網路上找學習的資源。

    剛開始是在 Linkedin Learning 學 Python，雖然內容還算理解，卻無從得到具體的職涯方向建議。無意間看到彭彭老師在台大開的課程後，嘗試先看 Youtube 教材。看過後覺得彭彭老師的教材深入淺出，幫助我建立程式語言的邏輯和概念，更理解到要成為一位軟體工程師，重視的是「耐心、細心、持續學習、獨立解決問題」的能力。現在自學3個月以來，持續在網路上找資源來學習，並且在VSC編寫看看是否呈現相同的效果，這種學習技術的方式，讓我感到腳踏實地。

    聽了彭彭老師在5/21線上招生說明會，思考老師說的「為什麼公司要錄取一個非本科系畢業的轉職工程師？」於是加深想要參加 WeHelp 的動機：「接受更有規劃的學習方針，訓練解決問題的能力，並且累積實作的專案」，提高自己轉職的成功率。

    我完全認同和理解彭彭老師所說的學習方式，因為我過去在學校和職場的經驗，也是靠著自己摸索、學習走出來的。大學念英文系，研究所跨到科技教育，出社會從事銷售工作做到管理職，一路上不斷嘗試錯誤才走到今天。對現在的我來說，全職投入轉職之前的準備，除了先自學確認自己有決心轉職的意願，另外，也說服全家人獲得他們的支持。

    希望成為 WeHelp的學員，以一年內轉職成為軟體工程師為目標，全力以赴！

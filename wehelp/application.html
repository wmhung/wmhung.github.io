自我介紹

我是洪暐銘，習慣大家叫我Bruce。10 年工作資歷，其中 8 年是負責軟體銷售業務，包含：美食外送服務、電商系統、行銷數據CDP、企業BPM系統，作爲業務主管帶領團隊時，曾為公司創造單月破千萬業績，YOY達160%的戰績。

做業務有高峰就有低谷，深刻體會收入不穩定、工作價值只能以業績高低來衡量，因此，決心想轉職軟體工程師，累積技術硬底子，追求穩定和發展潛力高的工作。

為了成為軟體工程師，曾做過什麼努力？若有具體專案作品請分享給我們。

（一）從網路上找付費、免費資源自學約 3 個月的時間，以下是我上過的課程：

  • 彭彭的課程（付費訂閱） - HTML, CSS, JavaScript
  • Linkedin Learning（年費會員） - Python, HTML, CSS, JavaScript
  • MDN Web Doc
  • HTML Standard
  • W3Schools Online Web Tutorial

    • （二）專案作品：只有本次提交的申請書

    如果參與這個計畫，會怎麼安排學習時間？

    已辭掉工作，全職投入。每天除了吃飯和睡覺，每日學習規劃如下：

  • 每天 9:00 - 9:30 復盤前一天學習狀況，如遇到自己無法解決的開發問題，至少花一天時間研究如何解決，無法自行解決，隔天詢問助教。
  • 每週一參加10:00 線上會議，並於會議結束後，開始學習課程。
  • 週二～四 9:30 - 18:00 按照課表學習，並於 18:30 花 15 分鐘撰寫每日學習報告，花 15 分鐘複習當日學習重點。
  • 每週五，參加實體課程，與同儕交流，有問題當面請教老師、助教。
  • 每週六～日，10:00 - 16:00練習當週學習的程式語言，並且進行該階段的作業或專案內容，同時嘗試累積更多side project。
    • 是否有想要加入的軟體公司？為什麼想加入該公司？
    想成為全端工程師（鑽研後端和ios應用程式開發），進入以開發「企業BPM」的軟體公司為目標，因為這類型公司規模小型大型都有，工作機會也多。。
    請描述一件讓你產生明顯負面情緒的事情，你如何處理該情緒？

    在工作上遇到無法以解決問題為目的的合作夥伴，會讓我很沮喪。遇到這種狀況，先以解決問題為優先，接下來再處理彼此溝通的問題。

    舉例來說，過去我在負責大型客戶專案期間，常遇到功能客製和系統串接問題，經常需要和後端工程師與客戶開三方技術會議。與會的工程師並非擅長面對客戶，因此會不經意地回答會過於仔細，偶爾發生會議越開越長，從原本討論範圍內延伸更多新的需求。

    此時，在會議上面對客戶，我會把焦點強調本次合作最大的價值和優先解決的問題，除非現場客戶端的窗口有權限改變專案內容，三方討論應該聚焦在原本討論範圍，其他需求也很重要可以放在之後階段討論。開完會後，我會先和工程師討論專案排程是否能如預期交給客戶，最後再主動提出自己能夠改進的地方，讓彼此合作可以更好，通常能讓雙方引起良性的互動。

    關於這份申請網頁，請分享一個你開發時的技術心得

    很有成就感！發現學習和實際進開發完全是兩回事，尤其是code寫進去後，網頁呈現卻沒反應或是改動到不對的地方，才知道有觀念沒搞懂。唯一解決的方式，就是要有耐心先找出哪裡有問題，再思考如何調整！

    這是我第一次開發網頁，彭彭老師曾提過「HTML不難，難是難在網頁切版」，結果實際開發果然被我遇到，因就因為沒搞清楚「容器和項目」的階層概念，例如：要調整項目中的文字、背景顏色、行寬、行高，都要在上一層容器的CSS選擇器設定，當段落越多，就越容易搞混自己現在到底改到哪一層了？？？

    後來，反覆觀看彭彭老師和Morten Rand-Hendriksen 課程，釐清切版的概念，同時找一個真實網站（WIRED）查看網頁原始碼編寫的架構，然後再回到VSC嘗試編寫．花了2天才解決問題。

    第一次申請。
    其他想要對我們說的事情？

    不論這次次申請是否錄取，都很感謝彭彭老師在 Youtube 上提供免費的入門課程，對於完全沒有程式語言基礎、又想在 37 歲時轉職當工程師的我來說，在自學過程中增加了不少信心，覺得程式語言的越來越有趣了。回顧自己的業務職涯，週遭的同事都在追求創業當老闆或者賺更多的錢。反觀自己隨著年資越深，在每次的客製專案內，總是想了解更多客戶的需求，協助客戶解決問題，而非單純追求成交。這讓我認真的思考，挑戰轉職工程師的路。

    思考轉職的過程中，我爬了不少文章、看過不少 Youtuber 影片，談論轉職工程師的心路歷程，都提到這是一份要持續學習、解決問題的工作，而且程式語言會持續推陳出新，不保持學習精進，就會逐漸失去競爭力。透過網路自學的過程中，一度像無頭蒼蠅，又很不喜歡坊間補習班的推銷手法，更是用盡可能尋找網路上可學習的資源，無意間看到彭彭老師在台大開的課程後，嘗試先看線上視頻。深深覺得，遇到會教的老師，可以加速學習的效率！

    現在自學3個月以來，謹記老師提過的「持續不斷學習，遇到問題嘗試解決！」，保持不斷在網路上找資源來學習，並且在VSC編寫看看是否呈現相同的效果，這種學習技術的方式，讓我感到腳踏實地。聽了彭彭老師在5/21線上招生會上提到一個很現實的問題：「為什麼公司要錄取一個非本科系畢業的轉職工程師？」聽完老師說明 WeHelp 的上課方式，我很確定這就是我要的。

    我完全能夠認同和理解彭彭老師所說的學習方式，因為我過去在學校和職場的經驗，也是靠著自己摸索、學習走出來的。大學念英文系，研究所跨到科技教育，出社會從事銷售工作做到管理職，一路上不斷嘗試錯誤才走到今天。對我來說，全職投入轉職的成本和風險很高，盤點下來我得放棄「目前能談到的年薪」和「投入半年的學習時間」。因此，我希望盡全力毫無保留的投入轉職，目前我得到家人的全力支持。

    希望能成為WeHelp的學員，以一年內轉職工程師為目標！

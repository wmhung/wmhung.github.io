    請描述一件讓你產生明顯負面情緒的事情，你如何處理該情緒？

    在工作上遇到無法以解決問題為目的的合作夥伴，會讓我很沮喪。遇到這種狀況，先以解決問題為優先，接下來再處理彼此溝通的問題。

    舉例來說，過去我在負責大型客戶專案期間，常遇到功能客製和系統串接問題，經常需要和後端工程師與客戶開三方技術會議。與會的工程師並非擅長面對客戶，因此會不經意地回答過於仔細，偶爾發生會議越開越長，從原本討論範圍內延伸更多新的需求。

    此時，在會議上面對客戶，我會把焦點強調本次合作最大的價值和優先解決的問題，除非現場客戶端的窗口有權限改變專案內容，三方討論應該聚焦在原本討論範圍，其他需求也很重要可以放在之後階段討論。開完會後，我會先和工程師討論專案排程是否能如預期交給客戶，最後再主動提出自己能夠改進的地方，讓彼此合作可以更好，通常能讓雙方引起良性的互動。

    關於這份申請網頁，請分享一個你開發時的技術心得

    發現學習和實際進開發完全是兩回事，尤其是 code 寫進去後，網頁呈現卻沒反應或是改動到不對的地方，才驚覺有觀念沒搞懂。我遇到的問題是彭彭老師提過的「網頁切版」，課程中「CSS 容器和項目」的概念是以項目水平排列當作案例，要調整項目中的文字、背景顏色、行寬、行高，都要在上一層容器的 CSS 選擇器設定。但是，製作這份網頁只會運用到容器和項目垂直排列，所以，水平排列的概念就不適用了。

    後來反覆觀看彭彭老師和 Morten Rand-Hendriksen（Linkedin Learning的線上講師）講解網頁切版的課程，釐清概念後，同時找一個真實網站（WIRED），查看網頁原始碼編寫的切版架構，確認自己是否能看懂，並且在 VSC 上實作幾次來驗證看看。

    重新開發時，為求編寫 HTML 時容易閱讀，避免眼花看錯行，因此將 CSS 獨立出來放置在 CSS 選擇器內。切版的部分，整理如下：

    1. 我將最外層容器 div 設置為 frame，設定整張網頁的底色為 lightgray；
    2. 進入下一層設置了 2 個項目垂直排列，一個 div 設置為 frametitle，設定了申請書大標題和黑色底色，另一個 div 設置為 frame2 設定整個內文的底色為 #eeee；
    3. 進入下一層為 9 個項目垂直排列，其中分為 2 個部分，第一部分是 8 個 div 設置為 headline ，底色設定為 gray，第二個部分是 1 個 div 設置為 footbottom，底色設定為 lightgray。其中每個 headline 各自包含 1 個 div 設置為 content，而 footbottom 內包含 1 個 div 設置為 footend；

    最後，用上述方式完成本次申請書的網頁切版。

    其他想要對我們說的事情？

    感謝彭彭老師在 Youtube 上提供免費的入門課程，對於完全沒有程式語言基礎、又想在 37 歲時轉職當工程師的我來說，在自學過程中增加了不少信心，覺得程式語言的越來越有趣了。回顧自己的業務職涯，週遭的同事都在追求創業當老闆或者賺更多的錢。反觀自己隨著年資越深，除了厭倦收入時高時低之外，在每次的案子，反而是越來越鑽研在協助客戶解決問題，而非單純追求成交賺獎金。因此，從一年前認真思考轉職工程師這條路。

    我擔心貿然地投入金錢和時間去上培訓班，會承擔過多的風險。於是先做一些功課瞭解這個職業，爬了不少文和看一些 Youtube 影片談論關於轉職工程師的心路歷程，也跟幾位工程師聊過，得到的反饋都是「這一份要持續學習、解決問題的工作，否則會很快失去競爭力」。為了確認自己是否有學習的意願和熱忱，從半年前，趁工作閒暇之餘，從網路上找學習的資源。

    剛開始是在 Linkedin Learning 學 Python，雖然內容還算理解，卻無從得到具體的職涯方向建議。無意間看到彭彭老師在台大開的課程後，嘗試先看 Youtube 教材。看過後覺得彭彭老師的教材深入淺出，幫助我建立程式語言的邏輯和概念，更理解到要成為一位軟體工程師，重視的是「耐心、細心、持續學習、獨立解決問題」的能力。現在自學3個月以來，持續在網路上找資源來學習，並且在VSC編寫看看是否呈現相同的效果，這種學習技術的方式，讓我感到腳踏實地。

    聽了彭彭老師在 5/21 線上招生說明會，思考老師說的「為什麼公司要錄取一個非本科系畢業的轉職工程師？」，這個問題加深我想要參加 WeHelp 的動機：「接受更有規劃的學習方針，訓練解決問題的能力，並且累積實作的專案」，提高自己轉職的成功率。

    我完全認同和理解彭彭老師所說的學習方式，因為我過去在學校和職場的經驗，也是靠著自己摸索、學習走出來的。大學念英文系，研究所跨到科技教育，出社會從事銷售工作做到管理職，一路上不斷嘗試錯誤才走到今天。對現在的我來說，全職投入轉職之前的準備，除了先自學確認自己有決心轉職的意願，另外，也說服全家人獲得他們的支持。

    希望成為 WeHelp 的學員，以一年內轉職成為軟體工程師為目標，全力以赴！
